Deep Learning Type Inference

Basic Information:

  • Title: Deep Learning Type Inference (深度学习类型推断)
  • Authors: Vincent J. Hellendoorn, Christian Bird, Earl T. Barr, and Miltiadis Allamanis
  • Affiliation: University of California, Davis (Vincent J. Hellendoorn)
  • Keywords: Type Inference, Deep Learning, Naturalness
  • URLs: Paper , GitHub

摘要：

  • a. 本文的研究背景：
    • 本文探讨了JavaSrcript和Python等动态类型语言中静态类型的挑战，并介绍了一个称为DeepTyper的深度学习模型，可以准确地预测变量和函数的类型注释。
  • b. 过去的方法、问题以及动机：
    • 本文强调了静态类型注释的重要性，特别是对于大型代码库项目。
    • 静态类型可以提高代码的健壮性和可维护性，但在JavaScript和Python等动态语言中的实现往往面临许多困难。
    • 过去的方法通常需要频繁的人工干预，工作量巨大且容易出现错误，限制了其在大规模项目中的应用。
    • DeepTyper的动机在于降低这一临界点，提供一种更加高效和准确的类型注释机制，并使得现有代码具备从静态类型检测中的好处。
  • c. 本篇文章提出的研究方法：
    • 本文提出DeepTyper模型，使用文本标记的层次结构来训练深度学习模型，捕捉各种context中类型的各种变体和关系。
    • 该模型能够推断出变量和函数的类型注释，同时可以充当编译器的交互工具，提高诊断效率。
    • 本文还提出了增强既有RNN模型的改进，以提高在函数长度一致性和其他一致性方面的效果。
  • d. 方法在任务中实现的性能：
    • DeepTyper提供了一种新的使用机器学习技术来提高代码静态类型可读性的方式。
    • 实现的结果是，DeepTyper的类型提示可以在在recall@50上达到准确率80％，并在一定的上下文中可以产生大约4000个类型注释，同时保持高于95％的准确率。接近人类程序员的准确率，克服了原有方法的瓶颈。

INTRODUCTION

现有的研究主要集中在程序设计语言的静态类型系统的优点，特别是在可维护性、可读性和错误检测方面的优势。有的研究通过用户研究显示静态类型语言如何提高代码的可维护性和可读性，还有的通过对开源项目的实证研究发现静态类型语言的错误发生率相对较低。

然而，一些最受欢迎的编程语言如Python和JavaScript采用的是动态弱类型系统，对这类语言进行类型推断往往要通过动态分析，比如Jalangi，这是因为特性如鸭子类型和JavaScript的eval()函数使得静态类型推断变得复杂。

而针对这一问题，有些语言如TypeScript提出了另一种解决方案，它们通过引入允许部分类型注解的类型系统来增强原有的语言，然后可以转译回原语言，这样TypeScript就可以在IDE中使用并编译，并可以转译成“纯”JavaScript，以在任何可以使用JavaScript的地方使用。这种方式降低了现有代码的类型化门槛，同时至少解锁了编译时类型检查的优点。

然而，从JavaScript迁移到TypeScript仍然具有挑战性，因为它需要对可能很大的现有代码库进行注解，这是一个耗时且可能出错的过程。本文提出的DeepTyper，利用深度学习对已有的类型代码进行学习，以推断JavaScript和TypeScript的新的类型注解。

以下是论文的主要贡献：

  1. 使用配对的语料库和可微的类型向量的类型推断学习机制，将离散类型分配任务放宽到连续的、实值向量函数。
  2. 展示了使用深度学习进行类型推断的潜力和挑战，特别是提出了对现有RNN的增强，增加了在文件中类型注解的一致性和准确性。
  3. 展示了概率学习器和合理的类型推断引擎之间的互惠互利的关系，可以相互提高性能。

总的来说，这篇论文的创新点是将深度学习应用于类型推断，特别是对JavaScript和TypeScript的类型推断，通过训练深度学习模型推断代码的类型。这种方法克服了静态类型推断的限制，并提出了一个概率学习器和类型推断引擎的相互增强方案，有望提高类型推断的性能。

为了达成这个目标，这篇论文主要提出了以下解决方案：

  1. 学习机制：这篇论文使用了一个新颖的学习机制，该机制使用一个对齐的语料库和可微分的类型向量，将离散类型的指派任务变得更为容易和连续。这样，类型推断任务就变成了一个连续的、实值向量函数的问题，从而使得深度学习模型可以更好地处理。
  2. 模型增强：作者提出了对现有的循环神经网络（RNN）的增强，以提高类型注解在一个文件中的一致性和准确性。这是对传统深度学习模型的一个重要改进，有助于改善类型推断的结果。
  3. 学习器和引擎的结合：这篇论文展示了如何将一个概率学习器（这里是深度学习模型）和一个类型推断引擎相结合，从而相互提高性能。这种结合使得类型推断既可以利用深度学习的强大推理能力，又可以利用类型推断引擎的准确性。

总的来说，这篇论文的主要贡献在于，它提出了一个新的、基于深度学习的类型推断方法，这种方法充分利用了深度学习的优势，同时克服了传统类型推断方法的一些限制。这种方法在实验中表现出了良好的效果，预计在未来的研究和应用中将具有广泛的应用前景。

背景：

  • a. 主题和特征：
    • 本文研究的主要内容为动态编程语言中的类型推断。
  • b. 历史发展：
    • 在动态编程语言中，如何实现类型注释是一个问题，它可以影响到代码的可读性和功能的可靠性，其注明类型信息的方法受到了人们的关注和研究。
  • c. 过去的方法：
    • 过去的方法主要是基于人工标注的方式，适用于小型代码库，但并不实用这种方式适用于大型的代码库项目。
  • d. 过去的研究不足：
    • 过去的方法面临着手动干预与时间成本高等问题，限制了其在大规模项目中的应用。
  • e. 目前需要解决的问题：
    • 如何更快更准确投入类型注释，使新旧开发团队快速渗透为类型检查，并无痛培养基于类型检查语言的意识。

方法:

  • a. 研究的理论基础:
    • 本文基于人工标注的层次结构来训练深度学习模型，捕捉各种contexts 中类型的各种变体和关系。
  • b. 本文的技术路线（逐步）:
    • 该模型采用的是基于DL的机器学习方法，从而达到从海量数据源中学习更准确的特定领域的语境。
    • 文章将深度学习应用于类型推断的领域，基于深度学习模型 DeepTyper 实现了如何有效地注释和类型检测 JavaScript 和 Python 等脚本语言，使用 LSTMs 和函数式编程等算法工具。
    • DeepTyper 可以在已知变量和函数的类型或函数定义及其上下文中指出其类型。
  • c. 技术路线的具体步骤（逐步）:
    • DeepTyper模型的核心是BiLSTM−CRF分类器，该分类器由前馈神经网络和基于隐马尔科夫模型的条件随机场组成，可以学习token序列中标记之间的关系。
    • 计算单元使用LSTM，双向循环神经网络模型旨在提取上下文数据，从而识别不同的type。
    • 深度网络的初始状态由通常的语言模型的预处理和适当的初始化过程给出。
    • 基于这个模型，可以为代码中的变量和函数添加静态类型，这将大大提高代码的可读性和可维护性。

作者受到POS和NER的启发，在这些任务中，机器学习模型需要从上下文中推断给定单词的角色。部分词性（Part-of-Speech，POS）标注和命名实体识别（Named Entity Recognition，NER）是自然语言处理（NLP）中的两个重要任务。

  1. 部分词性标注（POS）: 这是对文本中的每个单词进行分类和标注的过程，其分类和标注是基于每个单词在上下文中的语法功能和关系。例如，单词可以被标记为名词、动词、形容词、副词、连词等。举例来说，考虑句子 “The cat is on the mat.”。在此句中，“The”，“is”和“on”可以被标记为冠词，动词和介词，而“cat”和“mat”可以被标记为名词。
  2. 命名实体识别（NER）: 这是一个用于识别文本中命名实体的信息提取任务，例如人名、组织名、地点名、日期、时间等。举例来说，考虑句子 “John works for Microsoft in Seattle.”。在这个句子中，“John”是一个人名，可以被标记为PERSON，“Microsoft”是一个组织名，可以被标记为ORGANIZATION，“Seattle”是一个地点名，可以被标记为LOCATION。

这两种任务都需要理解文本的语境，以正确地执行标注。因此，这两个任务通常被用作训练和评估自然语言处理模型的基准。

DeepTyper的主要组成部分是递归神经网络（RNN），这种网络已经在许多自然语言注释任务中取得了广泛的成功，例如命名实体识别和机器翻译。RNNs是能够处理元素序列（如单词）的神经网络，因此非常适合进行类型推断任务。

模型的输入是一系列的标记（tokens），这些标记需要被转化为类型注释。为了模拟概率分布，我们定义了一个条件概率分布P(τ0…τN|s0…sN)，其中s_i表示输入序列中的元素，τ_i表示s_i的类型注释。在我们的案例中，注释是输入标记的类型。对于没有类型的标记（例如标点符号、关键字等），我们将它们与一个特殊的无类型符号进行对齐。

然而，仅仅使用RNN进行序列注释任务还不够，因为这种方法会忽视s_t右边的所有相关上下文，即s_{t+1}…s_N的信息。因此，作者使用了一种称为双向RNN（biRNN）的架构，它结合了两个运行方向相反的RNN：一个从前向后遍历序列，另一个从后向前遍历。

此外，DeepTyper还包含了一个一致性层。这个层级的目的是解决一种问题：一个变量可能在代码中多次使用，但其真实类型始终保持不变。如果我们忽视了多个标记之间的相互依赖性，我们的注释可能会在同一个变量的不同使用之间产生不一致性。一致性层通过将标记的上下文表示与所有绑定到相同标识符的标记的平均表示相结合，鼓励模型使用标识符所有用途的长距离信息。

实验的目标是为JavaScript和TypeScript代码从固定的词汇表中推荐有用的类型注解。其中，”有用的类型注解”被定义为开发者在TypeScript代码中手动添加的那些注解。在训练数据的生成过程中，这些注解被去除。

DeepTyper模型通过学习为文件中的每一个标识符出现分配一个概率分布（即类型向量）来学习提出这些推荐。在训练期间，模型的准确性以其生成的类型向量与真实的、独热编码的类型向量之间的交叉熵来度量。

在测试阶段，模型的任务是在开发者最初添加了类型注解（在生成对齐数据时这些注解被我们移除了）的位置推断出正确的注解。虽然模型为每个标识符的所有出现推断类型（因为这就是它的训练方式），但我们在原始类型注解上报告我们的结果，因为这是最现实的测试标准，也可以避免混淆。

Data collection

作者的数据收集流程如下：

  1. 收集数据：作者从Github上收集了最受欢迎的前1000个主要由TypeScript代码构成的开源项目，这个收集的时间点是2018年2月28日。这种方式与Ray等人的编程语言研究类似。
  2. 解析项目：作者使用TypeScript编译器tsc解析每个项目，该编译器为每个标识符的所有出现推断类型信息（可能是any类型）。
  3. 删除大文件：为了让序列能够在深度学习器的小批量(minibatch)中适应，删除了包含超过5000个令牌的所有文件。这只删除了少量的文件（约0.9%）和令牌（约11%）。
  4. 删除只含有TypeScript头文件的项目：这主要包括来自”DefinitelyTyped”生态系统的所有项目。

经过上述步骤，数据集中包含776个TypeScript项目。

训练集和测试集的定义如下：

  1. 训练集、验证集和测试集：作者将数据集随机按项目分成80%的训练数据，10%的验证数据（或保留数据）和10%的测试数据。
  2. 数据预处理：在训练分割上估计词汇表，并用通用的”UNKNOWN”令牌替换所有稀有的令牌（在我们的情况下，是那些出现次数少于10次的令牌）以及在验证和测试数据中未见过的令牌。
  3. 数据对齐：为创建一个原始和对齐的语料库，使用编译器为每个标识符添加类型注解。然后从TypeScript代码中移除所有类型注解，以创建更接近JavaScript代码的代码。
    • 全部标识符数据（训练）：在令牌和类型之间创建一个对齐的语料库，其中每个标识符的每次出现都有编译器的类型注解。这是用于训练的oracle数据类型。
    • GOLD, 只注解数据（测试）：我们只对齐开发人员注解的类型与添加注解的声明位置。所有其他的标记都与一个没有类型的标记对齐。这提供了开发人员关心的注解的最接近的近似，并作为我们的测试数据。

Metrics

准确性：模型对每个标识符出现的位置都会进行预测，每个预测的准确性将会被评估。这包括度量正确预测的排名，以及提取Top-K准确性指标。

一致性：作者还关心模型在代码中为标识符分配类型的一致性。理想情况下，DeepTyper对所有标识符在所有出现的位置分配的类型应该是一样的。但是，和所有的随机方法一样，DeepTyper并不总是这么精确。

Results

The researchers presented their results in three phases:

Baselines:

  • TypeScript编译器配合CheckJS工具运行，可以得到类型推断结果，作为另一个对比基准。

  • JSNice是一个利用大型语料库学习的依赖关系图统计数据进行JavaScript类型推断的方法。这个方法与DeepTyper是互补的，所以将它们的性能进行比较也是有意义的。

    [28] Veselin Raychev, Martin Vechev, and Andreas Krause. 2015. Predicting program properties from big code. In ACM SIGPLAN Notices, Vol. 50. ACM, 111–124.

Deep Learning for Type Inference: The researchers used deep learning models for type inference and showed how a simple RNN and DeepTyper performed on a test dataset. DeepTyper was shown to outperform a naive model that assigns a type to an identifier based on its type distribution at training time, achieving a top-1 accuracy of nearly 60% and a top-5 accuracy of over 80% across the dataset.

They also found DeepTyper to be more consistent, reducing the inconsistency rate to 15.4% from 17.3% in the plain RNN. Despite this, there are still challenges in predicting rarer types. The researchers suggest that this can be improved upon with a recommendation system that uses a confidence threshold to suggest types.

Conventional Type Inference: The researchers compared their results with the TypeScript compiler and CheckJS, finding that their model outperformed these in most cases. They found that developers typically add annotations where the type inference tool could not infer the correct type. By using a hybrid model, the researchers found that the model was able to correct many of the incorrect types and provide more precise type suggestions.

Comparison with JSNice: The researchers compared their model with JSNice, a tool that uses statistics from dependency graphs for type inference. Both DeepTyper and JSNice performed similarly, with each having their strengths and weaknesses. JSNice was precise when it did infer a type, but was often uncertain and didn’t provide a type. On the other hand, DeepTyper made more predictions but was incorrect more often than JSNice. A hybrid model was able to outperform both models individually.

总的来说，研究发现，深度学习模型DeepTyper在类型推断方面表现出色，表现优于简单的RNN以及常规的类型推断工具，如TypeScript编译器和CheckJS。该模型甚至可以与JSNice相媲美，后者采用不同的类型推断方法，显示了深度学习在这一领域的潜力。然而，仍有改进的空间，特别是在预测稀有类型以及模型倾向于更常见类型的问题上。

结论:

  • a. 开展此项工作的重要性:
    • DeepTyper 是一种用于指出不同类型的JavaScript和Python变量和函数注释的深度学习模型。
    • 本文的工作解决了静态类型推断的挑战，可以提供快速且准确的类型建议，有效地提高了代码库的可读性和可维护性。
    • 该模型的创新性表明了深度学习在静态类型推断中的应用前景。
  • b. 创新，性能和工作量:
    • 与基于规则的方法相比，DeepTyper可以自动处理更复杂的上下文依赖性。
    • DeepTyper以大约4000个文档注释为代价，实现了95％的准确率和50％的召回率。
    • DeepTyper 的结果表明了深度学习在程序分析中的有效性和价值，并为未来的机器学习研究提供了有用的启示。
  • c. 研究结论总结（列出要点）:
    • DeepTyper是一种基于深度学习的自动类型注释工具，能够为代码中的变量和函数提供准确和可靠的类型注释，并充当编译器的交互工具，提高代码审查工作的效率。
    • 在大规模项目的应用方便快捷，准确性也高于现有的静态类型推断方法，其提供了深度学习在程序分析中的成功应用示范。

Typescript和Javascript的区别

TypeScript 和 JavaScript 是密切相关的编程语言，它们之间的主要区别在于 TypeScript 在 JavaScript 的基础上增加了静态类型系统。

以下是 TypeScript 和 JavaScript 之间的一些主要区别：

  1. 类型系统：JavaScript 是一种动态类型语言，意味着你可以在运行时改变变量的类型。另一方面，TypeScript 是一种静态类型语言，这意味着你需要在编写代码时指定变量的类型。这可以帮助在编译阶段捕获许多类型相关的错误。
  2. 类和接口：TypeScript 支持接口和类，这使得使用面向对象编程 (OOP) 技术更加容易和直观。虽然最新的 JavaScript 版本 (ES6 及以后的版本) 也支持类，但不支持接口。
  3. 编译：TypeScript 代码需要被编译成 JavaScript 代码才能在浏览器或其他 JavaScript 运行环境中运行。这个编译过程也被称为 “转译”。这使得 TypeScript 可以使用未来的 JavaScript 功能，并在当前的 JavaScript 环境中运行。
  4. 模块：TypeScript 提供了更高级的模块化功能，使得大型代码库的管理和维护变得更容易。
  5. 工具支持：由于 TypeScript 的类型系统，许多代码编辑器和 IDE 可以提供更好的自动完成、导航和重构功能。

总的来说，尽管 TypeScript 需要额外的编译步骤，并且学习曲线可能比 JavaScript 稍微陡峭一点，但许多开发者发现 TypeScript 提供的静态类型检查和其他特性对于构建大型、复杂的应用程序非常有帮助。
